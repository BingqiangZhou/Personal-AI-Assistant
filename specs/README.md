## 使用说明

### 创建新需求
1. 复制模板：`cp templates/requirement-template.md active/[模块名]/requirement.md`
2. 根据模板填写需求内容
3. 创建对应的任务跟踪文档：`cp templates/task-tracking-template.md active/[模块名]/tasks.md`
4. 在下表中更新需求列表

### 查找需求
- 查看下方的需求索引表
- 使用文件系统搜索功能
- 通过Git历史追踪变更

### 更新需求
1. 直接编辑对应的Markdown文件
2. 记录变更日期和原因
3. 更新任务跟踪文档
4. 提交代码变更

## 模块说明

### User Management (用户管理)
- 用户注册、登录、认证
- 权限管理
- 个人资料管理
- 安全设置

### Knowledge Base (知识库)
- 文档管理
- 知识组织
- 搜索功能
- 标签系统

### Assistant (AI助手)

### Subscription (订阅管理)

### Multimedia (多媒体处理)

## 工作流程提示

1. **需求分析阶段**
   - 使用requirement-template.md创建需求文档
   - 明确用户故事和验收标准
   - 分解任务并分配责任人

2. **开发阶段**
   - 使用task-tracking-template.md跟踪进度
   - 每日更新任务状态
   - 记录遇到的问题和解决方案

3. **验证阶段**
   - 使用verification-template.md进行验收测试
   - 记录所有发现的问题
   - 确保所有验收标准满足
